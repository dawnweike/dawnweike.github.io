Windows基础

一、系统目录

windows –系统安装目录

program files –程序安装目录

program files（x86）这个出现代表这个电脑是64位的

programData –程序的临时数据，隐藏文件（要记得打开显示中隐藏项目选项）

perflogs–windows7日志文件

（1）windows

​ windows目录下重要的文件目录System32

​ System32目录中含有配置信息，DOS文件，其中drivers文件目录里的etc文件里有hosts（来影响网站的解析），访问优先级高于DNS服务器。

​ System32其中的config文件里有SAM文件，所有用户的登录名和密码都储存在这个文件里。!

（2）用户

​ 文件含有用户的所有信息，以及操作信息，其中用户的文件里面的桌面信息，可能跟直接用目录下桌面信息内容有所不同，用身份西安的目录内容更全。还有用户目录里面的文档和下载页挺重要的。

二、计算机服务

服务是一种应用程序类型，它在后台运行。

​ 服务应用程序通常可以在本地和通过网络为用户提供一些功能，例如客户端 / 服务器应用程序、 Web服务器、数据库服务器以及其他基于服务器的应用程序。

（1）查看计算机本地服务

运行里面输入service.msc，点击确定

（2)常见的服务

web 服务：一种基于互联网的分布式系统，旨在让不同的设备和系统可以通过互联网互相交换和共享信息。

dns 服务（分布式数据库系统）：一个分布式数据库系统，用于将人类的易于记忆的域名转换为计算机易于理解的IP地址。

dhcp 服务（计算机网络协议）：用于自动分配 IP 地址和其他网络配置信息的计算机网络协议。

功能包括：

  1. 自动为连接网络的客户端设备分配 IP 地址。
  2. 提供主机名、DNS 服务器的地址以及域名的信息。
  3. 允许网络管理员通过集中化的方式管理多个设备的网络配置。

邮件服务：是通过网络传送信件、单据、资料等电子信息的通信方法

telnet 服务（远程登录协议）：一种远程登录协议，属于TCP/IP协议族的组成部分，主要用于实现互联网上的远程登录服务。它允许用户在本地计算机上通过Telnet程序与远程主机的控制台交互，类似于直接操作该主机的物理控制台。使用Telnet时，用户需要输入用户名和密码以验证身份，并通过Telnet服务器发送和接收数据。

ssh 服务：一种应用于应用层的网络协议，主要用于实现远程登录、执行命令或文件传输等功能。它是专为远程管理和网络服务提供的安全性协议，旨在确保数据在网络传输过程中不被未经授权的用户读取。SSH协议通过加密通信方式，结合公钥加密和对称密钥加密等方式，为远程计算机提供了一种安全访问方法。

ftp 服务（文件传输协议）：FTP服务的主要功能包括文件的上传（上传文件到FTP服务器）、下载（从FTP服务器下载文件到本地计算机）以及其他文件操作，如修改、删除等。FTP协议能够确保数据传输的可靠性，因为它是面向连接的，即在客户端和服务器端建立连接之前需要进行“三次握手”的过程。此外，FTP还提供了对用户权限的管理功能，使得不同用户可以根据需要设置不同的访问权限。

smb 服务：即Server Message Block服务，是一种网络协议，主要用于在计算机网络中共享文件、打印机和其他资源。它是客户端-服务器型的协议，允许多台计算机共享这些设备和数据。SMB协议广泛应用于Windows操作系统中，而Samba是一个开源项目，旨在使Linux和UNIX系统能够通过SMB协议与其他系统共享文件和打印机。此外，Samba还支持将Linux或UNIX机器集成到Windows域环境中，作为域控制器（Domain Controller）参与Active Directory等服务。

三、计算机端口

• 端口的作用是用来区分服务
• 端口不可以重复使用
• 端口范围是从 1-65535 （ 1-1024分给了系统自带的一些服务木马病毒一般使用高位端口。）
• 根据端口，我们可以探测计算机开放的服务、操作系统、网络角色。

四、常见的端口

• HTTP 协议代理服务器常用端口号： 80/8080/3128/8081/9080
• FTP （文件传输）协议代理服务器常用端口号： 21
• Telnet （远程登录）协议代理服务器常用端口： 23
• TFTP （ Trivial File Transfer Protocol ），默认的端口号为 69/udp ；
• SSH （安全登录）、 SCP （文件传输）、端口重定向，默认的端口号为 22/tcp ；
• SMTP Simple Mail Transfer Protocol (E-mail) ，默认的端口号为 25/tcp （木马 Antigen 、 Email
• Password Sender 、 Haebu Coceda 、 Shtrilitz Stealth 、 WinPC 、 WinSpy 都开放这个端口）；
• POP3 Post Office Protocol (E-mail) ，默认的端口号为 110/tcp ；
• TOMCAT ，默认的端口号为 8080 ；
• WIN2003 远程登陆，默认的端口号为 3389 ；
• Oracle 数据库，默认的端口号为 1521 ；
• MS SQL*SERVER 数据库 server ，默认的端口号为 1433/tcp 1433/udp ；
• QQ ，默认的端口号为 1080/udp

五、注册表

• 注册表是 windows 操作系统中的一个核心数据库，其中存放着各种参数，直接控制着 windows 的启动、硬件驱动程序的装载以及一些 windows 应用程序的运行，从而在整个系统中起着核 心作用。这个作用包括了软、硬件的相关配置和状态信息，比如注册表中保存有应用程序和资源管理器外壳的初始条件、首选项和卸载数据等，联网计算机的整个系统的设置和各种许可，文件扩展名与应用程序的关联，硬件部件的描述、状态和属性，性能记录和其他底层的系统状态信息，以及其他数据等。

（1）注册表结构

运行里输入 regedit 点击确定就可以注册表了![1707201991182]

• 1.HKEY_CLASSES_ROOT
• 管理文件系统。根据在 Windows 中安装的应用程序的扩展名 , 该根键指明其文件类型的名称，相应打开该文件所要
调用的程序等等信息。
• 2**.HKEY_CURRENT_USER**
管理系统当前的用户信息。在这个根键中保存了本地计算机中存放的当前登录的用户信息 , 包括用户登录用户名
和暂存的密码。在用 户登录 Windows 98 时，其信息从 HKEY_USERS 中相应的项拷贝到 HKEY_CURRENT_USER 中。
• 3.HKEY_LOCAL_MACHINE
• 管理当前系统硬件配置。在根键这个中保存了本地计算机硬件配置数据 , 此根键下的子关键字包括在 SYSTEM.DAT 中 ,
用来提供 HKEY_LOCAL_MACHINE 所需的信息 , 或者在远程计算机中可访问的一组键中。
• 这个根键里面的许多子键与 System.ini 文件中设置项类似。
• 4.HKEY_USERS
• 管理系统的用户信息。在这个根键中保存了存放在本地计算机口令列表中的用户标识和密码列表。同时每个用户的预
配置信息都存储在 HKEY_USERS 根键中。 HKEY_USERS 是远程计算机中访问的根键之一。
• 5**.HKEY_CURRENT_CONFIG**
• 管理当前用户的系统配置。在这个根键中保存着定义当前用户桌面配置 ( 如显示器等等 ) 的数据 , 该用户使用过的文档
列表（ MRU ），应用程序配置和其他有关当前用户的 Windows 98 中文版的安装的信息。

*注意：

HKEY _LOCAL MACHINE\SOFTWARE\MicrosoftWindows\currentVersion\Run

这个是开机启动程序放置位置，容易中木马病毒。

如果真中了开机自启病毒，上述此目录下没有找到此木马，注意：此文件下HKEY_LOCAL_MACHINE\SOFTWARE\WOW6432Node\Microsoft\Windows\CurrentVersion\Run

七、powershell 命令

​ Powershell 是运行在 windows 机器上实现系统和应用程序管理自动化的命令行脚本环境!

（1）如何运行程序、脚本和已有的软件

• 在系统路径下运行程序、脚本、批处理文件或者其它可执行文件，可以直接输入文件名运行。比如我把一个 Test.cmd 文件放在了” C:\Windows\System32” 下
• 如果文件名字或路径中有空格，需要用（ ‘ ）引号将命令扩起来，同时在前面加上符号（ & ）
• 要运行当前目录下的命令，需要文件名前加 .\

（3）POWERSHELL 常用命令

• Get-command 获取命令
• Get-help get-service 获取命令使用说明
• Get-Host 或 $PSVersionTable 获取 PowerShell 版本信息

八、域环境配置

• 域控制器
• 域控制器中包含了由这个域的账户、密码、属于这个域的计算机等信息构成的数据库。当电脑联入网络时，域控制器首先要鉴别这台电脑是否是属于这个域的，用户使用的登录账号是否存在、密码是否正确。如果以上信息有一样不正确，那么域控制器就会拒绝这个用户从这台电脑登录。不能登录，用户就不能访问服务器上有权限保护的资源，他只能以对等网用户的方式访问 Windows 共享出来的资源，这样就在一定程度上保护了网络上的资源。
